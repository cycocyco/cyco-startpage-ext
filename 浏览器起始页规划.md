# Edge New Tab Extension - Spec

## Environment
- browser: Edge (Chromium MV3)
- type: extension, chrome_url_overrides.newtab
- location: W:\自用插件\浏览器扩展\起始页\
- deployment: local only, no store publish
- interaction: mouse only, no keyboard shortcuts

## Modules

### Clock
- format: 24h HH:mm:ss
- date: YYYY-M-D 星期X (e.g., 2026-7-16 星期四)
- position: top-left

### Weather
- api: https://api.open-meteo.com/v1/forecast (free, no key)
- params: latitude, longitude, current=temperature_2m,apparent_temperature,relative_humidity_2m,weather_code
- WMO weather code -> icon/text (内置映射表，纯 emoji 或内联 SVG，不依赖外部图标服务)
- **定位决策（Q3）：首次询问一次**。首次打开请求 geolocation，坐标存 `storage.geo`；之后读缓存不再弹窗。用户拒绝 → fallback 北京 (39.9042,116.4074) 并写空标记避免反复打扰
- display: temperature, feels-like, humidity, icon
- layout: compact, 与 clock 同行的右上角，无外框/无背景卡（纯文字+图标，融入背景）
- 降级：geolocation 被拒 / 请求失败 → 显示最后缓存 or 隐藏该模块，不阻塞其它模块

### Search
- engines: Bing(default) / Google / Baidu, switchable (选择持久化到 storage)
- autocomplete: use each engine's suggest API
  - Bing: https://api.bing.com/qsonhs.aspx?type=cb&q=
  - Google: https://suggestqueries.google.com/complete/search?client=chrome&q=
  - Baidu: https://suggestion.baidu.com/s?wd=
- **补全方案决策（Q1）**：先做「Phase 0 实测」——用最小 fetch 探测三接口是否返回 `Access-Control-Allow-Origin: *`
  - 能跨域 → 用 fetch + 300ms debounce + 递增 seq 竞态丢弃，正常补全
  - 不能跨域 → 降级为「无补全，直接回车搜索」
  - JSONP 仅作为最后 fallback（需在 manifest 配 content_security_policy 放宽，优先级最低）
- 补全下拉：键盘虽非主交互，但保留上下键+回车可选中（增强可用性）；鼠标点击直接搜索
- domain detection: regex /^[a-zA-Z0-9][a-zA-Z0-9.-]+\.[a-zA-Z]{2,}$/ -> direct navigation (补 http:// 前缀)
- open mode: current tab (window.location.href)
- 安全：补全词做 textContent 写入，禁止 innerHTML 防 XSS

### Bookmarks
- source: chrome.bookmarks.getTree() (Edge favorites)
- add flow: click "add bookmark" button -> tree modal -> user selects -> save to chrome.storage.local
- tree modal 规格（落实重点，规划原描述偏简）：
  - 递归渲染 getTree() 多层目录，目录可折叠展开（默认展开前两层）
  - 仅叶子节点（url 存在的书签）可选中；目录节点不可选（点击=展开/折叠）
  - 顶部内置搜索框，按标题/url 实时过滤树（不区分大小写）
  - 支持多选（checkbox），一次保存多条
  - 已存在于收藏列表的项标记「已添加」不可重复选
- **手动添加（Q4）**：树形 modal 内额外提供「手动添加」表单（标题 + URL 输入，URL 做格式校验），与树选择并存；手动添加项存 `manual:true` 标记
- no grouping, flat list
- favicon auto fetch: https://www.google.com/s2/favicons?domain={hostname}&sz=64 (sz 指定尺寸，避免模糊)
- favicon 渲染（2026-07-17 修复）：用 `new Image()` + JS `onload/onerror` 监听，**禁止内联 `onerror` 属性**（MV3 扩展 CSP 会拦截内联事件，导致图标静默失败）；加载失败 → 回退显示标题首字母（暖白），不破图、不丢布局
- open mode: current tab
- edit/delete: 每条 hover 显示删除按钮；无编辑（仅增删）
- defaults: YouTube, Bilibili, GitHub, MC百科(https://www.mcmod.cn)
- no reset-to-defaults button

### Background Color
- dark mode only, no light toggle
- color picker + preset swatches
- supports solid colors and gradients
- no custom image upload
- **入口：右上角齿轮按钮 ⚙ 弹出的小设置面板内**（不常驻展开，原型确认）

### Hot Search (Baidu Trending)
- **模块决策（Q2）：可开关**。storage 键 `hotsearchEnabled`（默认 true），设置面板提供开关；关闭后整模块不请求、不渲染
- **数据源实测结论（2026-07-17）**：
  - ✅ `https://dabenshi.cn/other/api/hot.php?type=baidu` —— **唯一稳定可用**，返回 `{success,title,data:[{index,title,desc,...}]}`，42KB JSON
  - ❌ UAPI / 韩小韩(vvhan) / DailyHot 公共实例 / oioweb / tenapi 等 —— 全部返回空 / HTML 页面 / 502（免费第三方热搜 API 现网基本已失效）
  - **结论：热搜源就用 dabenshi.cn 单源，不做多源切换**（其它源都挂了，切换也是空）
- api: https://dabenshi.cn/other/api/hot.php?type=baidu
- refresh: every 5 hours (缓存用 chrome.storage.local 存 {ts, data}，键名 `hotsearch_v2`)
- **缓存复发防护（2026-07-17）**：热搜图曾因 manifest 未放行 `bcebos.com` 图片域名而全部失败。修复=在 host_permissions 加 `*.bcebos.com`；并将缓存键从 `hotsearch` 升到 `hotsearch_v2` + 启动时 `remove("hotsearch")`，使旧缓存失效、强制重新拉取干净数据；图片 onerror 加一次 cache-bust 重试兜底
- display: top 15, 大卡片网格（参考 Edge 原生 new tab 卡片风格，重做排版）
  - **排版层级（2026-07-17 重设计）**：卡片含顶部图片区(16:9) + 下方「序号 + 标题」；序号用衬线大字（clay 色，前 3 名 accent 陶土色），标题 15px / 行高 1.5 / 字重 500，两行截断，hover 标题转纯白
  - 图片优先用 api 返回的 pic 字段（JS `Image` 加载，失败/无图 → 渐变占位块 + emoji 兜底，**不用内联 onerror**，因 MV3 CSP 禁止内联事件）
  - **图片域名白名单（关键）**：pic 来自 `*.cdn.bcebos.com` / `*.bcebos.com`，必须在 manifest `host_permissions` 声明，否则扩展拦截图片请求（2026-07-17 修复：之前热搜图全失败即因此）
  - 宽屏 4 列（更大卡片），中屏 3 列（≤1100px），窄屏 2 列（≤680px）；卡片 hover 上浮 + 暖褐阴影 + 边框转 accent
  - **布局解耦（2026-07-17）**：热搜区 `.hot-wrap` 最大宽 1280，脱离上方模块（搜索/书签限宽 760 居中），热搜卡片可更大、不与上面对齐，视觉更有层次
- image 失败 fallback：渐变占位 + emoji（不破图）
- offline / 请求失败 / 接口 5xx：改为显示上次缓存（若有）；完全无缓存时显示一行「热搜暂时不可用」占位

### RSS News Module (可选增强)
- **定位**：热搜之外可选的「我订阅的新闻/博客」模块，用 RSS 标准接入（用户自提供订阅链接）
- **现成可用源实测（2026-07-17）**：
  - ✅ `https://sspai.com/feed` —— 少数派（效率工具/数字生活，默认推荐）
  - ✅ `https://www.ruanyifeng.com/blog/atom.xml` —— 阮一峰（技术随笔）
  - ✅ `https://www.solidot.org/index.rss` —— 奇客Solidot（科技快讯）
  - ✅ `https://www.36kr.com/feed` —— 36氪（创业/商业科技）
  - ❌ `https://www.zhihu.com/rss` —— 知乎已不提供
- 默认订阅：少数派；切换器样式复用搜索引擎那种「大框包胶囊按钮」
- 解析：用 `DOMParser` 解析 RSS/Atom XML，取 `<item>`/`<entry>` 的 title+link
- 状态：规划阶段，未排入 Build Order（待热搜/书签稳定后可选做）
- open mode: current tab

## Manifest & Permissions
- manifest_version: 3
- chrome_url_overrides.newtab: "newtab.html"
- permissions: ["storage", "bookmarks"]
- host_permissions (CORS 白名单，fetch/JSONP 跨域必需):
  - "https://api.open-meteo.com/*"
  - "https://api.bing.com/*"
  - "https://suggestqueries.google.com/*"
  - "https://suggestion.baidu.com/*"
  - "https://www.google.com/s2/favicons/*"
  - "https://dabenshi.cn/*"
- 说明：JSONP 本身不受 host_permissions 限制，但加上更稳；Open-Meteo / favicon 走 fetch 必须声明

## Persistence & Defaults (持久化与默认值)
- chrome.storage.local 存储键：
  - `bg`: { type: 'solid'|'gradient', value: string }  (默认深色纯色 #1A1813)
  - `engine`: 'bing'|'google'|'baidu'  (默认 bing)
  - `bookmarks`: [{id, title, url, manual?}]  (默认 4 条；手动添加项 manual:true)
  - `geo`: {lat, lon} | null  (Q3 定位缓存；null=未授权用北京)
  - `hotsearchEnabled`: boolean  (Q2 默认 true)
  - `hotsearch`: { ts: number, data: [...] }  (缓存)
- **读取策略（Q5）：仅页面加载时读一次，不做 chrome.storage.onChanged 监听**（简单优先；多标签页不同步，刷新即生效）
- 首次安装：storage 为空时写入默认值（不弹窗）
- 数据损坏兜底：读取后 JSON.parse 失败 / 字段缺失 → 回退默认值，不白屏
- 无 reset-to-defaults 按钮（规划已定），但书签可逐条删除恢复默认需重装（已记录在限制内）

## Visual Direction Decision (视觉方向)
- 选定：**深色暗色风（Dark Warm Organic）**。理由：起始页是浏览器常驻页，深色更护眼、沉浸感强，且与 `--card-bg: rgba(45,42,36,0.85)` 一致。
- 同目录参考文件 `warm-organic-hard-prompt.md` 的 REQUIRED 模板默认是**浅色纸面风**（卡片 `bg-[#E8DED1]`）。在深色模式下做如下**对应改写**（保留其禁止项与核心理念，仅翻转明度）：
  - 卡片底色由浅沙 `#E8DED1` → 深色半透 `rgba(45,42,36,0.85)` / `#2D2A24`
  - 边框由 `#D4BFA5` 暖米 → `#4A453C` 暖褐（保持暖调，不降到冷灰）
  - 文字由 `#2D2A24` 深褐 → `#F5F0EB` 暖白
  - 强调色、圆角、软阴影、禁止项（冷灰/玻璃态/纯黑/霓虹蓝）**完全沿用** hard-prompt
- 落地前自检必须同时过两份文档的「禁止清单」与「自检清单」。

## Color System (Warm Organic Dark)
Reference files: CSS参考.md, warm-organic-hard-prompt.md

CSS variables:
--bg: user-selectable (solid or gradient, dark tones, 上限不纯黑，最低明度用 #1A1813 类暖黑)
--card-bg: rgba(45,42,36,0.85)  (深色半透，非 glassmorphism——无 backdrop-blur)
--card-bg-solid: #2D2A24
--border: #4A453C
--border-soft: rgba(212,191,165,0.4)  (暖米描边，用于图片/分隔)
--text: #F5F0EB
--text-dim: rgba(245,240,235,0.65)  (次级文字，不用冷灰 #999)
--accent: #C86A4A (Terracotta, focus/emphasis)
--secondary: #7A8B5E (Olive)
--warm-shadow: 0 4px 20px -4px rgba(45,42,36,0.10)  (深色下叠加暖褐阴影)
--warm-shadow-hover: 0 8px 30px -6px rgba(45,42,36,0.15)

Rules (沿用 hard-prompt + 深色改写):
- border-radius: 8-10px (rounded-lg equivalent)，禁止 rounded-xl/2xl/3xl/full（小圆点除外）
- shadows: 暖褐 rgba(45,42,36) 半透明，禁止冷灰/纯黑 shadow
- 禁止冷灰 #999 / #6B7280 等中性色；中性色必须偏暖
- 禁止 backdrop-blur / glassmorphism
- 禁止纯黑 #000 作正文（上限 #2D2A24 深褐；背景最低 #1A1813 暖黑）
- 禁止渐变文字 background-clip:text、禁止单侧 accent 粗边框装饰、禁止嵌套卡片
- card hover: translateY(-2px) + deeper shadow + 可选 active:scale-[0.98]
- input focus: border -> accent + focus ring rgba(200,106,74,0.20)
- 动效必须有 prefers-reduced-motion 备选

## Layout & Responsive (布局自适应) — 定稿 v3
- **整体结构（原型确认定稿）**：分散全宽纵向流
  - 顶部：Clock(top-left) + Weather(top-right)，同一行**水平居中对齐**（非顶部对齐）
  - 其下：Search 搜索框（hero，自身宽度居中），含引擎切换（Bing/Google/Baidu 胶囊按钮，包在一个大框内，当前项高亮 accent）
  - 再下：Bookmarks 书签网格，占满容器宽度，自然铺开
    - **书签卡片缩小**：最小列宽 84px，紧凑内边距/图标/字号
    - **折叠行为**：默认只显示第一行，底部渐隐遮罩；下方「展开全部 ▾ / 收起 ▴」按钮；当书签数 ≤ 一行容量时按钮自动隐藏
    - 判定基于「一行容量 = floor(容器宽 / 卡片最小宽)」，非高度判定（避免宽屏一排放下导致按钮误隐藏）
  - 再下：HotSearch 热搜卡片网格（大卡片、带图、参考 Edge 原生），占满宽度，标题左对齐分区
  - 各模块间留 36px 间距，整体靠上自然流
  - 设置：右上角固定**齿轮按钮 ⚙**，点击弹出小面板（背景色板 + 热搜开关），不常驻展开
  - 容器：max-width ~1100px 居中，整体下移（顶部 ~14vh 留白），视觉重心偏下
- 视口断点：
  - 宽屏 ≥1100px：Bookmarks auto-fill（minmax 84px），HotSearch 5 列
  - 笔记本 768–1099px：HotSearch 3 列
  - 竖屏/窄屏 <768px：HotSearch 2 列，Bookmarks 2–3 列
- 模块间距用 CSS 变量（桌面 36px），统一响应式
- 避免内容溢出：HotSearch 卡片数固定 15，超出容器滚动而非挤压

## Accessibility (可访问性)
- 正文对比度 ≥ WCAG AA 4.5:1：--text #F5F0EB on --card-bg 达标；--text-dim 仅用于非必要次级信息
- 所有可点击元素有 :focus-visible 焦点环（accent 色），键盘可达（虽主交互为鼠标，仍保留 focus 样式）
- 热搜卡片图片加 alt / 失败 fallback 为纯色占位（不破图）
- 动效统一包在 @media (prefers-reduced-motion: reduce) 内禁用 transform/animation
- 天气/热搜加载与失败状态有明确的文本提示（不止图标）

## Build Order
Phase 0 - 实测 (Q1 决策前置):
0. 最小 fetch 探测脚本，验证 Bing/Google/Baidu suggest 三接口 CORS（Access-Control-Allow-Origin）。结论决定 Search 补全走 fetch / 降级 / JSONP

Phase 1 - Skeleton:
1. manifest.json (MV3, chrome_url_overrides.newtab + storage + bookmarks + host_permissions)
2. newtab.html (6-module layout skeleton, 含响应式断点骨架, 极简无品牌字 Q6)
3. newtab.css (dark warm-organic theme via CSS variables, 含 prefers-reduced-motion)
4. Load unpacked extension via edge://extensions

Phase 2 - Core:
5. Clock (JS realtime update, 24h + weekday date)
6. Search (3-engine switch + 补全[按 Phase0 结论] + domain detection + debounce/竞态, current-tab nav)
7. Bookmarks (chrome.bookmarks.getTree -> 树形选择 modal(折叠/搜索/多选) + 手动添加表单(Q4) -> chrome.storage.local persist, 增删, favicon 自动获取+失败兜底)
8. Weather (首次定位存 geo Q3 -> Open-Meteo API -> 内置 WMO 映射 -> compact render + 失败降级)
9. Hot search (dabenshi.cn fetch, 5h chrome.storage 缓存, top-15 3-column cards, 失败用缓存/占位; 受 hotsearchEnabled 开关 Q2 控制)

Phase 3 - Polish:
10. Background color picker (solid + gradient presets + color picker, 持久化)
11. 设置面板：hotsearchEnabled 开关 (Q2)、手动添加入口 (Q4)
12. Bookmark tree modal 完整 UI (搜索/多选/已添加标记/手动表单)
13. Extension icons (16/48/128) — 可用脚本批量生成（见 Risks）
14. Transition/fade animations (wrap in prefers-reduced-motion)

## Risks & Open Questions (风险与待定)
- **API 稳定性（实测 2026-07-17）**：免费第三方热搜 API 现网几乎全失效（UAPI/韩小韩/DailyHot/oioweb/tenapi 均空或 502）；仅 dabenshi.cn 可用。已定为单源，但仍有随时失效风险 → 保留 hotsearchEnabled 关闭开关兜底
- **JSONP 安全**：补全接口返回第三方脚本，注入前无法做 CSP 校验；MV3 扩展默认 CSP 较严，JSONP <script> 注入可能受限 → 优先尝试 fetch（已加 host_permissions），JSONP 作 fallback
- **geolocation 权限**：newtab 页请求定位，用户可能拒绝；需明确 fallback 到北京且不再反复打扰
- **图标生成**：三个 PNG 可用 ImageMagick / sharp / 在线工具批量生成，风格用 accent #C86A4A 陶土底 + 暖白图标
- **Edge 书签 API 兼容性**：Edge 兼容 chrome.bookmarks.*，但中文「收藏夹」根节点名不同，树渲染需对根节点做泛化处理
- **风格漂移自检**：交付前同时过 hard-prompt 与 CSS参考.md 两份清单（圆角/阴影/边框/禁止项）
